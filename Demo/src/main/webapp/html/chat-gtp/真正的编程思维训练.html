# 🛠 2. 每阶段配套实战任务 + 提升练习

---

## 【阶段①】状态思维训练

### 🎯目标：

学会用状态机替代流程控制，理清系统中一切状态变迁

### ✅ 任务1：写一个红绿灯控制器

* 输入：时间、按钮触发
* 输出：红-绿-黄交替逻辑
* 限制条件：用**状态表**+状态转移而非`if-else`

```js
const stateMap = {
RED: { timer: 'GREEN' },
GREEN: { timer: 'YELLOW' },
YELLOW: { timer: 'RED' }
};
```

### ✅ 任务2：UI 表单的可编辑状态机

* 状态：初始、编辑中、提交中、成功、失败
* 转换：按钮触发/验证失败
* 要求：可视化状态图（如 XState 图），同时写代码执行

---

## 【阶段②】内存与性能感知训练

### 🎯目标：

理解“内存布局=性能”，体会 CPU 缓存与结构优化

### ✅ 任务1：链表 vs 数组性能对比

* 实验：访问 100 万元素并计时
* 工具：Node.js 或 Python
* 输出：实际耗时对比报告 + 原因解释

### ✅ 任务2：写一个结构优化的数据聚合器

* 场景：大量订单记录需做统计
* 初版：对象数组 + for循环聚合
* 高级版：预分组结构 +索引缓存 +减少内存跳跃

---

## 【阶段③】抽象压缩力训练

### 🎯目标：

学习将 `逻辑分支` 变为 `数据驱动决策`

### ✅ 任务1：写一个 RPG 战斗行为决策系统（表驱动）

* 输入：敌人状态、我方状态
* 输出：行动指令（逃跑、攻击、技能）
* 构造决策表代替`if else`

### ✅ 任务2：构建一个规则驱动表单渲染器

* 输入：配置 JSON
* 输出：渲染组件（如 input/select）
* 你不能写 if(type==='input')，而要做组件注册器

---

## 【阶段④】副作用控制训练

### 🎯目标：

感受纯函数系统如何减少复杂度与错误

### ✅ 任务1：写一个不可变用户状态更新系统

* 输入：用户对象 + 修改指令
* 要求：输出新的对象，原对象不动
* 工具：自己手写或用 [Immer.js](https://immerjs.github.io/immer/)

### ✅ 任务2：构建一个时间旅行调试器

* 模拟 Redux 状态，每次更新存入快照
* 可通过箭头“后退/前进”回到任意状态

---

## 【阶段⑤】可调试性思维训练

### 🎯目标：

写出“不会丢状态、能追踪历史”的代码

### ✅ 任务1：为异步表单写日志系统

* 每一步（打开页面、输入字段、提交、失败）写入日志
* 日志包含：时间戳、状态快照、触发事件
* 可导出 JSON，方便复盘

### ✅ 任务2：为一个多状态组件加“状态播放器”

* 示例：对话机器人、小游戏角色
* 能点击“回放”看每个状态过渡（可结合 Canvas 或控制台）

---

## 【阶段⑥】解释器与 DSL 训练

### 🎯目标：

构建一个能“读取规则并执行”的小语言系统

### ✅ 任务1：写一个 JSON 表达式解释器

```json
{ "op": "+", "left": 3, "right": { "op": "*", "left": 2, "right": 4 } }
```

* 输出：11
* 原理：将 JSON 转 AST，然后用递归 `eval()`

### ✅ 任务2：写一个行为 DSL 的运行器

* 定义语法：用户点击、等待、跳转
* 执行逻辑：根据规则数组逐条运行行为

```json
[
{ "action": "click", "target": "button1" },
{ "action": "wait", "ms": 2000 },
{ "action": "navigate", "to": "/success" }
]
```

---

## 【阶段⑦】系统涌现与容错设计

### 🎯目标：

从“构建功能”转向“管理崩溃与压力”

### ✅ 任务1：构建一个聊天后端模拟系统

* 功能：发送消息、用户在线、离线
* 添加：消息重试、延迟队列、写失败自动恢复（基于本地队列模拟）

### ✅ 任务2：为一个小服务构建熔断机制

* 每次请求模拟 80% 成功率
* 加入：

* 熔断器（连续失败 n 次后断开）
* 恢复机制（等待冷却再重试）
